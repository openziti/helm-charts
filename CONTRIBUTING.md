# Contributing

NetFoundry welcomes all and any contributions. All open source projects managed by NetFoundry share a common
[guide for contributions](https://docs.openziti.io/policies/CONTRIBUTING.html).

If you are eager to contribute to a NetFoundry-managed open source project please read and act accordingly.

## Agent Guidelines

When using AI coding agents (for example with `CLAUDE.md`, `GEMINI.md`, or similar agent files), align your
agent instructions with this contributor guide.

- Batch related operations together.
- Prefer exploration over asking. Try an approach and report results.

### Chart Documentation

Chart `README.md` files are generated by `helm-docs` from two sources:

1. `values.yaml`: inline comments prefixed with `# --` become the parameter table.
   Follow the existing convention:

   ```yaml
   # -- Description of the parameter.
   parameterName: defaultValue
   ```

1. `README.md.gotmpl`: Go template for prose sections (description, prerequisites, usage examples, deprecation
   banners). This file controls the overall README structure; `helm-docs` injects the values table where
   `{{ template "chart.valuesTable" . }}` appears.

Never edit chart `README.md` directly. It is overwritten by `helm-docs`.

After changing `values.yaml` comments or `README.md.gotmpl`, regenerate docs:

```bash
./generate-docs.bash
```

The script iterates over `charts/ziti-*`, `charts/zrok`, and `charts/zrok2`.
When adding a new chart, add its glob to the loop in `generate-docs.bash`.

### Labeler Workflow

When adding or removing a chart in `charts/`, update `.github/labeler.yml` to keep `notes:<chart-name>` labels
in sync with the chart list.
